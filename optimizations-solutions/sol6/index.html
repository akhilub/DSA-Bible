    Sol6

    The optimization algorithm calculates a minimum total cost of $18,103k. The five depots 1, 5, 8, 9, and 12 are built. The following table details how the customers are delivered to from these depots. All depots built, except depot 5 are used to their maximum capacity.

    Table 6.4: Delivery plan

    1 – 5 75 100 – – – – – – – 120

    9 – – – – 110 – – – – 65 95 –
    12 – – – – – – 90 60 30 – – –

    Explanation

    To formulate the mathematical model, we write \(DEM_c\) for the demand by customer (sales center) \(c\), and \(CAP_d\) for the maximum capacity of depot \(d\). The fixed cost of constructing depot \(d\) is given by \(CFIX_d\), the cost of delivery from depot \(d\) to customer \(c\) as \(COST_{dc}\). Furthermore, let \(DEPOTS\) be the set of all possible depot locations and \(CUST\) the set of customers to be delivered to from these depots.

    To solve the problem, we need to know which depots will be opened. So we define binary variables \(build_d\) that take the value \(1\) if site \(d\) is chosen, and otherwise \(0\).

    In addition, we also need to know which depot(s) deliver(s) goods to a customer. We introduce variables \(fflow_{dc}\) for the fraction of the demand of customer \(c\) that is satisfied from depot \(d\). These variables take their values in the interval [0, 1]; we therefore have the constraints (6.1).

    \[\forall d \in DEPOTS, c \in CUST : fflow_{dc} \leq 1 \tag{6.1}\]

    The demand of every customer needs to be entirely satisfied:

    \[\forall c \in CUST : \sum_{d \in DEPOTS} fflow_{dc} = 1 \tag{6.2}\]

    We now have to model the fact that the total amount leaving depot d must be no more than its capacity CAPd, but that the flow is zero if the depot is not built. The constraints (6.3) capture this.

    \[\forall d \in DEPOTS: \sum_{c \in CUST} DEM_c \cdot fflow_{dc} \leq CAP_d \cdot build_d \tag{6.3}\]

    Let us now see why. Since \(fflow_{dc}\) is the fraction of customer \(c’s\) satisfied demand, \(fflow_{dc} \cdot DEM_c\) is the amount going from \(d\) to \(c\). The total outflow from \(d\) must be no more than \(CAP_d\) if \(d\) is built (\(build_d\) = 1), and must be 0 if \(build_d\) = 0.

    The total cost to be minimized consists of the costs for constructing the depots and the delivery costs. These are the two sums comprising the objective function (6.4).

    The complete mathematical model is given by the following.

    Model Formulation

    \[minimize \sum_{\forall d \in DEPOTS} CFIX_d \cdot build_d + \sum_{\forall d \in DEPOTS} \sum_{\forall c \in CUST} COST_{dc} \cdot fflow_{dc} \tag{6.4}\]
    \[\forall d \in DEPOTS, c \in CUST : fflow_{dc} \leq 1 \tag{6.5}\]
    \[\forall c \in CUST : \sum_{d \in DEPOTS} fflow_{dc} = 1 \tag{6.6}\]
    \[\forall d \in DEPOTS: \sum_{c \in CUST} DEM_c \cdot fflow_{dc} \leq CAP_d \cdot build_d \tag{6.7}\]
    \[\forall d \in DEPOTS, c \in CUST : fflow_{dc} \geq 0 \tag{6.8}\]
    \[\forall d \in DEPOTS : build_d \in \mathbb{N} \tag{6.9}\]
    Algorithm Implementation
    • The problem is a mixed integer programming (MIP) problem where we need to minimize the total cost of fulfilling demand from different depots.
    • We define binary variables x[j] to indicate whether depot j is open (1) or not (0).
    • We define continuous variables y[j][i] to represent the fraction of demand i fulfilled by depot j. These are constrained between 0 and 1.
    • The objective function is to minimize the total cost, which includes both fixed costs for opening depots and variable costs for fulfilling demand.
    • Constraints ensure that each demand is fully met by exactly one depot and that the total demand served by an open depot does not exceed its capacity.
    • After defining the model and constraints, we optimize it and check the status to determine if an optimal solution was found.

    Solutions

    import gurobipy as gp
from gurobipy import GRB
import os

def solveTotalMinCostForDepot(DELIVERY_COST_D2C, DEMAND, CAPACITY, FIXED_COST):

    D = len(DELIVERY_COST_D2C)
    C = len(DELIVERY_COST_D2C[0])

    m = gp.Model("depotLocation")

    TOTAL_COST = m.addVar(vtype=GRB.CONTINUOUS, lb = 0, obj = 1, name = "TOTAL_COST")

    x={}
    for j in range(D):
        x[j+1] = m.addVar(vtype=GRB.BINARY,obj=1, name="x_{}".format(j+1))

    y={}
    for j in range(D):
        for i in range(C):
            y[j+1,i+1] = m.addVar(vtype=GRB.CONTINUOUS, lb=0, ub=1, obj=1, name="y_{},{}".format(j+1,i+1))

    m.update()
    #m.modelSense = GRB.MINIMIZE
    m.setObjective(TOTAL_COST, GRB.MINIMIZE)

    for i in range(C):
        m.addConstr(gp.quicksum(y[j+1,i+1] for j in range(D)) == 1)

    for j in range(D):
        m.addConstr(gp.quicksum(DEMAND[i]* y[j+1,i+1] for i in range(C)) <= (CAPACITY[j]*x[j+1]))

    m.addConstr((gp.quicksum(FIXED_COST[j]*x[j+1] for j in range(D)) + (gp.quicksum(DELIVERY_COST_D2C[j][i]*y[j+1,i+1] for i in range(C) for j in range(D)))) == TOTAL_COST)

    m.optimize()

    if m.status == GRB.status.OPTIMAL:
        #m.write("depotLocation.lp")
        #os.system("imshow depotLocation.lp")
        #m.write("depotLocation.sol")
        #os.system("imshow depotLocation.sol")
        print(f"\nOptimal total cost: ${TOTAL_COST.X:.2f}")

        print("\nDelivery Plan Matrix (Depot -> Customer in Tons):\n")

        # Print header row with customer indices
        header = ["C{:02d}".format(i+1) for i in range(C)]
        print("       ", "\t".join(header))

        # Print each depot's deliveries to customers
        for j in range(D):
            row = []
            for i in range(C):
                amount = y[j+1, i+1].X * DEMAND[i]  # actual tons delivered
                row.append("{:.1f}".format(amount))
            print("D{:02d} :".format(j+1), "\t".join(row))

    elif m.status == GRB.status.INFEASIBLE:
        m.computeIIS() # IIS tells you which constraints, when removed, makes the m feasible
        #m.write('inf.ilp')

    print('Total Minimum Cost= $%g' % m.objVal)
